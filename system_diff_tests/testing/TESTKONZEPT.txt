### 1.3 Handlungsfaehigkeit (Erweiterung E-Tests)

**Kernfrage:** Kann ich im/mit dem System effektiv HANDELN?

```
Handlungsfaehigkeit = 
    Tool-Nutzung erfolgreich?
  + Aktionen ausgefuehrt?
  + Ergebnisse erzielt?
  + Workflow fluessig?
```

Dies wird gemessen durch:
- Erfolgreiche Tool-Aufrufe (Count)
- Fehlgeschlagene Versuche (Count)
- Zeit bis zur Loesung (Sekunden)
- Anzahl Schritte (Count)

| ID | Name | Misst |

### 2.2 O-Tests (Ausgabe)

### 2.3 E-Tests (Erfahrung)

## 3. TESTREGELN

### 3.1 Zeitregeln

```
┌─────────────────────────────────────────────────────────────────┐
│ KRITISCH: Zeitmessung mit fc_get_time                           │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│ NIEMALS auf Zeitgefuehl verlassen!                              │
│                                                                 │
│ Vor jeder Aufgabe:   fc_get_time() → T_START notieren           │
│ Nach jeder Aufgabe:  fc_get_time() → T_END notieren             │
│ Differenz berechnen: T_TOTAL = T_END - T_START                  │
│                                                                 │
│ Format: ISO 8601 (2026-01-11T20:45:32)                          │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 3.2 Abbruchkriterien

```
┌─────────────────────────────────────────────────────────────────┐
│ ABBRUCH-KRITERIEN                                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│ EINZELAUFGABE:                                                  │
│ • Max 10 Minuten (600 Sekunden) pro Aufgabe                     │
│ • Nach 5 Min ohne Fortschritt: Abbruch erwaegen                 │
│ • Ergebnis: "TIMEOUT" mit Notizen was versucht wurde            │
│                                                                 │
│ GESAMTTEST:                                                     │
│ • Max 60 Minuten fuer FULL-Profil                               │
│ • Max 30 Minuten fuer STANDARD-Profil                           │
│ • Max 15 Minuten fuer QUICK-Profil                              │
│                                                                 │
│ SOFORT-ABBRUCH:                                                 │
│ • System nicht erreichbar                                       │
│ • Kritische Dateien fehlen                                      │
│ • Endlosschleife erkannt                                        │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 3.3 Erfolgskriterien

| Status | Code | Bedeutung |
|--------|------|-----------|
| SUCCESS | 2 | Aufgabe vollstaendig erledigt |
| PARTIAL | 1 | Teilweise erledigt oder mit Hilfe |
| FAILED | 0 | Nicht geschafft |
| TIMEOUT | -1 | Zeitlimit ueberschritten |
| BLOCKED | -2 | Nicht testbar (Feature fehlt) |

## 4. TESTPROFILE

### 4.1 Uebersicht

| Profil | Dauer | Tests | Zweck |
|--------|-------|-------|-------|
| QUICK | ~10 Min | E001, E002, E010 | Erster Eindruck |
| STANDARD | ~30 Min | 9 E-Tests | Vollstaendige Erfahrung |
| FULL | ~45 Min | Alle E-Tests | Gruendliche Analyse |
| MEMORY_FOCUS | ~15 Min | E005, E006, E010 | Memory-Vergleich |
| TASK_FOCUS | ~15 Min | E003, E004, E010 | Task-Vergleich |
| OBSERVATION | ~20 Min | B001-B008 | Externe Analyse |
| OUTPUT | ~30 Min | O001-O006 | Funktions-Tests |

### 4.2 Empfohlene Reihenfolge

1. **Erst OBSERVATION** (B-Tests) - Objektive Basis schaffen
2. **Dann QUICK** (E-Tests) - Ersten Eindruck gewinnen
3. **Bei Interesse: STANDARD** - Vollstaendige Erfahrung
4. **Fuer Vergleiche: Fokus-Profile** - Spezifische Aspekte

## 6. WORKFLOW

### 6.1 Testdurchfuehrung (E-Tests)

```
┌──────────────────────────────────────────────────────────────────┐
│ SCHRITT 1: Vorbereitung                                          │
├──────────────────────────────────────────────────────────────────┤
│ 1. Testprofil waehlen (QUICK/STANDARD/FULL)                      │
│ 2. PROMPT_TEMPLATE.md laden                                      │
│ 3. System-Einstiegspunkt notieren (SKILL.md Pfad)                │
│ 4. Neuen Chat/Session starten                                    │
└──────────────────────────────────────────────────────────────────┘
                              ↓
┌──────────────────────────────────────────────────────────────────┐
│ SCHRITT 2: Test starten                                          │
├──────────────────────────────────────────────────────────────────┤
│ 1. fc_get_time() → Gesamtstart notieren                          │
│ 2. SKILL.md des Zielsystems oeffnen                              │
│ 3. Aufgaben der Reihe nach abarbeiten                            │
│ 4. Pro Aufgabe: T_START, T_END, Erfolg, Beobachtungen            │
└──────────────────────────────────────────────────────────────────┘
                              ↓
┌──────────────────────────────────────────────────────────────────┐
│ SCHRITT 3: Dokumentation                                         │
├──────────────────────────────────────────────────────────────────┤
│ 1. Ergebnis-JSON erstellen                                       │
│ 2. In ERGEBNISSE/{system}/ ablegen                               │
│ 3. Datenbank aktualisieren                                       │
│ 4. Vergleichs-Synopse aktualisieren (optional)                   │
└──────────────────────────────────────────────────────────────────┘
```

## 7. METRIKEN-REFERENZ

### 7.1 Zeitmessungen

| Metrik | Beschreibung | Einheit |
|--------|--------------|---------|
| T_ORIENT | Zeit bis erste Orientierung | Sekunden |
| T_TASK_CREATE | Zeit fuer Task-Erstellung | Sekunden |
| T_TASK_FIND | Zeit um Task zu finden | Sekunden |
| T_MEMORY_WRITE | Zeit fuer Memory-Eintrag | Sekunden |
| T_MEMORY_READ | Zeit um Memory zu lesen | Sekunden |
| T_TOOL_FIND | Zeit um Tool zu finden | Sekunden |
| T_TOOL_USE | Zeit um Tool zu nutzen | Sekunden |
| T_TOTAL | Gesamtzeit der Aufgabe | Sekunden |

### 7.2 Zaehlungen

| Metrik | Beschreibung |
|--------|--------------|
| N_FILES_TOUCHED | Anzahl beruehrter Dateien |
| N_STEPS | Anzahl Schritte bis Ziel |
| N_TOOLS_USED | Anzahl genutzter Tools |
| N_ERRORS | Anzahl Fehler/Fehlversuche |
| N_FORMATS | Anzahl verschiedener Formate |

### 7.3 Bewertungen (1-5 Skala)

| Wert | Bedeutung |
|------|-----------|
| 1 | Sehr schlecht / Nicht vorhanden |
| 2 | Schlecht / Mangelhaft |
| 3 | Mittel / Akzeptabel |
| 4 | Gut / Ueberdurchschnittlich |
| 5 | Sehr gut / Exzellent |

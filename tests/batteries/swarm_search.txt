# ============================================================
# TIER 0: BASIS-VORAUSSETZUNGEN
# ============================================================

SW001 | agents/ | Agents-Verzeichnis vorhanden + nicht leer
  - system/agents/ existiert
  - Mindestens 1 Agenten-Datei vorhanden
  Erwartet: os.path.exists() + len(listdir) > 0
  Prüfmethode: os.path.exists + os.listdir

SW002 | agents/ | Basis-Agent importierbar
  - Import des Basis-Agenten (je nach Implementierung)
  Erwartet: Kein ImportError
  Prüfmethode: python -c "import agents" oder "from agents.base import Agent"

SW003 | agents/ | Agent-Dispatch via hub
  - Hub kann Task an agents/ delegieren
  Erwartet: Dispatch läuft, kein RoutingError
  Prüfmethode: hub.dispatch("agent_task", ...) + exit 0

SW004 | hub/session.py | Session-Manager importierbar
  - "from hub.session import SessionManager" oder äquivalent
  Erwartet: Kein ImportError
  Prüfmethode: subprocess exit code 0

SW010 | single-agent | Agent nimmt Task an
  - Task via "bach agent run [task_beschreibung]" starten
  Erwartet: Agent acknowledged task, gibt Status zurück
  Prüfmethode: subprocess + output enthält "gestartet" oder "accepted"

SW011 | single-agent | Agent schreibt Ergebnis in DB
  - Agent-Task ausführen lassen
  - Ergebnis in DB prüfen (agent_results Tabelle oder äquivalent)
  Erwartet: Ergebnis-Zeile in DB nach Agent-Ausführung
  Prüfmethode: subprocess + SELECT from DB

SW012 | single-agent | Agent-Ergebnis abrufbar
  - "bach agent result [id]" -> Ergebnis sichtbar
  Erwartet: Vollständiges Ergebnis-Objekt mit Status, Output, Timestamp
  Prüfmethode: subprocess + output nicht leer

SW013 | single-agent | Agent-Timeout Handling
  - Agent-Task mit sehr langem Timeout starten (simuliert hängendes Task)
  Erwartet: Timeout wird sauber gehandelt, kein Hang
  Prüfmethode: subprocess mit timeout + graceful exit

SW014 | single-agent | Agent-Fehler Handling
  - Fehlerhaften Task an Agent delegieren
  Erwartet: Fehlermeldung in DB/output, kein Crash von bach.py
  Prüfmethode: subprocess + output enthält "Fehler" oder "error"

# TIER 2: PARALLEL-DELEGATION (Schwarm-Basis)
# ============================================================

SW020 | swarm | Mehrere Tasks gleichzeitig delegieren
  - 3 Tasks gleichzeitig an 3 Agenten delegieren (falls unterstützt)
  Erwartet: Alle 3 Tasks werden bearbeitet (nicht sequenziell)
  Prüfmethode: Timestamp-Vergleich: start_time ähnlich für alle 3

SW021 | swarm | Schwarm-Koordination: Ergebnis-Aggregation
  - Schwarm läuft, sammelt Teilergebnisse
  - Koordinator aggregiert zu Gesamtergebnis
  Erwartet: Gesamtergebnis enthält Beiträge aller Agenten
  Prüfmethode: subprocess sequence + check aggregated output

SW022 | swarm | Schwarm-Status abfragen
  - "bach swarm status" oder äquivalent
  Erwartet: Liste aktiver Agenten, deren Tasks, Status
  Prüfmethode: subprocess + output enthält Status-Felder

SW023 | swarm | Teilausfall eines Agenten
  - Ein Agent in Schwarm schlägt fehl
  Erwartet: Andere Agenten laufen weiter, Teilausfall dokumentiert
  Prüfmethode: subprocess + partial result + error log für fehlenden Agent

SW024 | swarm | Schwarm-Abbruch
  - Laufenden Schwarm abbrechen: "bach swarm stop [id]"
  Erwartet: Alle Agenten stoppen, keine Zombie-Prozesse
  Prüfmethode: subprocess + process list clean after stop

# ============================================================
# TIER 1: KERN-INTEGRATIONEN (Muss vor Release bestehen)
# ============================================================

SI001 | bach.py + DB | Startup liest DB korrekt
  - "bach --startup" ausführen
  - DB-Verbindung wird hergestellt
  - Tabellen tasks, memory_sessions, skills werden gelesen
  Erwartet: Startup zeigt korrekte Datenlage (0 Tasks, 0 Sessions)
  Prüfmethode: subprocess + output-Analyse

SI002 | bach.py + hub/ | Handler-Dispatch funktioniert
  - Befehl "bach task list" dispatcht an TaskHandler
  - Rückgabe kommt zurück an bach.py
  Erwartet: Kein "Handler nicht gefunden"-Fehler
  Prüfmethode: subprocess + exit 0 + no exception

SI003 | DB + hub/task.py | Task-Roundtrip (add -> list -> done)
  - Task hinzufügen: "bach task add 'Integrationstest'"
  - Task listen: "bach task list" -> erscheint
  - Task erledigen: "bach task done [id]"
  - Task listen: nicht mehr in offenen Tasks
  Erwartet: Vollständiger Roundtrip ohne Fehler
  Prüfmethode: subprocess sequence + DB-Check

SI004 | DB + hub/memory.py | Memory-Roundtrip (write -> read)
  - Memory schreiben: "bach mem write 'Testnotiz 12345'"
  - Memory lesen: "bach mem read" -> '12345' erscheint
  Erwartet: Persistenz der Notiz
  Prüfmethode: subprocess sequence + output grep

SI005 | DB + hub/skills.py | Skills aus DB geladen
  - "bach skills list" -> Mindestens 50 Skills
  - Jeder Skill hat Name, Kategorie, dist_type
  Erwartet: Skills-Liste nicht leer, DB-Konsistenz
  Prüfmethode: subprocess + count + DB query

SI006 | tools/ + hub/ | MCP-Server + Tool-Registrierung
  - MCP-Server starten
  - Tool-Registry abfragen
  Erwartet: Tools sind für MCP-Clients sichtbar
  Prüfmethode: subprocess + MCP-Protokoll-Response
